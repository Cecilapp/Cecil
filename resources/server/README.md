# Server resources

This directory contains helper files used by Cecil's local PHP server.

## Files

- `router.php`: router for PHP built-in server (`php -S`), used to serve the generated site from the output directory.
- `livereload.js`: script injected into HTML responses to enable automatic browser reload during local development.

## How it works

- `router.php` resolves requested paths, serves static files, and handles custom 404 behavior.
- For HTML pages, it injects `livereload.js` when available.
- `livereload.js` listens to `/watcher` using Server-Sent Events and reloads the page when changes are detected.

## Local runtime artifacts

The router may create/read these files during development:

- `changes.flag`: temporary flag used to notify reload events.
- `server.log`: request log generated by the local server.
- `headers.ini`: optional custom response headers by path.

These files are runtime artifacts and are not part of the source code.
